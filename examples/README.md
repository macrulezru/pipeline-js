# Examples

Focused, copy-pasteable snippets for patterns that come up often but don't fit
neatly into the main [README](../README.md) walkthrough. Each file is
self-contained and commented.

These import from `"rest-pipeline-js"` as a consumer would (after `npm install
rest-pipeline-js`) — they are not run as part of this repo's test suite or
build. For a runnable, interactive app instead, see [`demo/`](../demo).

| File | Shows |
| --- | --- |
| [`pagination-fanout.ts`](./pagination-fanout.ts) | Fanning out many paginated requests in parallel with a `concurrency` cap, instead of opening hundreds of connections at once. |
| [`edge-fetch-adapter.ts`](./edge-fetch-adapter.ts) | A custom `HttpAdapter` built on native `fetch`, for Cloudflare Workers / Deno / other environments without Node's `axios`. |
| [`sse-stream.ts`](./sse-stream.ts) | A `StreamStageConfig` step consuming a Server-Sent Events endpoint chunk by chunk. |
| [`redis-cache-store.ts`](./redis-cache-store.ts) | A `CacheStore` backed by Redis, so cached responses are shared across multiple server instances instead of living in one process's memory. |
| [`redis-rate-limiter-store.ts`](./redis-rate-limiter-store.ts) | A `RateLimiterStore` backed by Redis, so a rate limit is shared across server instances instead of each one enforcing its own (which multiplies the real-world limit by the instance count). |
| [`redis-circuit-breaker-store.ts`](./redis-circuit-breaker-store.ts) | A `CircuitBreakerStore` backed by Redis, so open/closed/half-open state is shared across server instances instead of each needing its own `failureThreshold` failures before opening. |
| [`opentelemetry-tracing.ts`](./opentelemetry-tracing.ts) | W3C `traceparent` header generation plus a `TracingProvider` hook (duck-typed against OpenTelemetry's `Span` API) for wiring in a real tracing backend, including correlating requests with a pipeline's `runId`. |
| [`idempotent-mutations.ts`](./idempotent-mutations.ts) | Sending an `Idempotency-Key` header on mutating requests, manually or auto-generated by `RequestExecutor` across retry attempts. |
